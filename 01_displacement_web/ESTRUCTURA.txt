=================================================================================

KEY DIRECTORIES EXPLAINED:
=================================================================================

00_predictive_displacement_model/
    Core machine learning pipeline for training and evaluating models.
    Contains all notebooks for data analysis, model training, and reporting.
    Stores trained models, metrics, and visualizations.

01_displacement_web/
    Production web application with Flask backend and React frontend.
    Provides interactive interface for making predictions using trained models.
    Connects to Colombian RUV database for data validation.

db/ folders (both locations)
    Storage for trained models, encoders, scalers, and preprocessing artifacts.
    CRITICAL: Models in 01_displacement_web/db/ must be downloaded separately
    from Google Drive (not in git due to size constraints).

fig/ folders
    All generated visualizations, plots, and performance reports.
    Includes confusion matrices, ROC curves, feature importance, etc.

=================================================================================

IMPORTANT NOTES:
=================================================================================

1. MODEL FILES NOT IN GIT:
   - All .pkl, .keras, and large model files are in .gitignore
   - Download separately from Google Drive links in README.md

2. REQUIRED ENVIRONMENTS:
   - Training: Python 3.13 (conda: model_training)
   - Web App Backend: Python 3.10 (conda: displacement)
   - Web App Frontend: Node.js v24.11.1 LTS

3. DATA SOURCE:
   - Colombian Unique Registry of Victims (RUV)
   - Accessed via Socrata API (datos.gov.co)
   - 7+ million victim records

4. EXECUTION ORDER FOR TRAINING:
   001 → 002 or 003 (parallel) → 004
